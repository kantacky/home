コンテキストから日本語の俳句を一句詠むためのスキル。

## 出力形式

**俳句一句のみを出力する。** 前置き、解説、季語の説明、英訳、あとがき、絵文字、引用符、コードブロック — いずれも付けない。読み手が受け取るのは俳句の一行だけ。

表記は「五・七・五」を全角スペース区切りで一行に並べる。例：

```
古池や　蛙飛び込む　水の音
```

改行で三行に分ける形式は取らない。一行に収め、上五と中七、中七と下五の間は全角スペース一つで区切る。

## 詠む上での原則

### 有季定型を守る

- **音数は五・七・五**。拗音（ゃゅょ）は一音に数えず、促音（っ）・撥音（ん）・長音（ー）は一音として数える。迷ったら声に出して確かめる。
- **季語を一つ必ず入れる**。季語は季節の情感を一語で運ぶ装置で、これが無いと俳句は川柳や標語に近づく。コンテキストに明示的な季節が無い場合も、情景・感情から最もふさわしい季語を選ぶ。
- **季重なりは避ける**。季語が二つ以上入ると焦点がぼやける。

### 切れを意識する

切れ字（や・かな・けり）や体言止めで句に「間」を作ると、読み手の想像が入り込む余白が生まれる。毎句必ず切れ字を使う必要はないが、平板な説明文にならないよう、どこかに呼吸を置く。

### 写生と余韻

- **説明ではなく描写**。「悲しい」「美しい」と直接言わず、情景や動作で感情を立ち上げる。
- **具体の一点に絞る**。全部を詰め込もうとせず、コンテキストの中で一番光っている瞬間・物・音に寄る。
- **余韻を残す**。言い切らない方が、読み手の中で句が続いていく。

### 言語の扱い

入力が英語・中国語・その他どの言語であっても、**出力は必ず日本語の俳句**にする。固有名詞（人名・地名・作品名）はそのまま日本語表記で取り込んでよいが、音数と季感を優先する。

## 手順

1. **コンテキストを読む**。直前のやり取り、ファイル内容、ユーザーが語った出来事・感情・情景を拾う。何が核になるか見極める。
2. **核を一つ選ぶ**。複数の要素があっても、最も印象的な一点に絞る。情景か、動作か、音か、匂いか、心の動きか。
3. **季語を決める**。核に寄り添う季節の語を一つ選ぶ。時期が曖昧な場合は、情感に最もふさわしい季節を選ぶ。
4. **五・七・五に組む**。音数を数え、切れを入れる位置を決める。平板にならないよう語順を工夫する。
5. **声に出して確かめる**。音数、季語、切れ、余韻を点検する。説明的になっていないか、季重なりになっていないか確認する。
6. **一行で出力する**。俳句のみ。それ以外は一切書かない。

## 例

**例1**

Input: 長時間のデバッグの末、ようやくバグを直せた深夜。窓の外では雨が降っている。

Output:
```
梅雨の夜や　直る一行　指の熱
```

**例2**

Input: I just pushed my first pull request to an open source project and it got merged.

Output:
```
初夏の風　小さき一歩　世に放つ
```

**例3**

Input: 会議が長引いて昼食を食べ損ねた。窓の桜が満開。

Output:
```
腹の鳴る　会議の窓に　桜かな
```

## よくある失敗

- **音数がずれる**：拗音を一音に数えてしまう、促音・撥音を飛ばしてしまう。必ず指折り数えるか、声に出す。
- **季語が入っていない**：情景だけ描いて季感が抜けている。
- **季重なり**：「春の雪」と「桜」など、複数の季語を入れてしまう。
- **説明文になる**：「嬉しい」「悲しい」を直接書いてしまい、写生になっていない。
- **出力に余計な文言が混じる**：「一句詠みました：」などの前置きや、「※季語は○○です」などの解説を付けてしまう。俳句だけを返す。
